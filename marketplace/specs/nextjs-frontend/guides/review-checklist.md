# Review Checklist

Before handoff, check:

- Route files follow App Router conventions.
- Server/client component boundaries are intentional.
- New content is typed and locale variants stay aligned.
- SEO metadata, canonical, alternates, and structured data were considered for
  route changes.
- Public form inputs are validated and sanitized.
- Secrets are not exposed through client environment variables.
- UI works on mobile and desktop.
- Visible UI follows the project root `DESIGN.md` when present.
- Lint, type check, build, and any local tests/checks were run.
- Missing automated coverage or manual-release blockers are called out.
